## Overview

This design introduces a **dynamic custom field resolution layer** that maps schema field names (snake_case) to Asana custom field GIDs at runtime using the task's existing `custom_fields` list. The approach eliminates static GID placeholders, removes environment-specific coupling, and provides self-documenting schemas while maintaining type safety and zero additional API calls.

---

## Requirements Summary

This design addresses [PRD-0003.1](../requirements/PRD-0003.1-dynamic-custom-field-resolution.md), which defines requirements for dynamic custom field resolution.

| Category | Key Requirements |
|----------|-----------------|
| **Normalization** | FR-NORM-001 to FR-NORM-008: snake_case to/from Title Case, ALL CAPS, hyphenated, etc. |
| **Resolution** | FR-RESOLVE-001 to FR-RESOLVE-006: Lookup by normalized name, session caching, GID override |
| **Caching** | FR-CACHE-001 to FR-CACHE-004: Thread-safe caching, per-session cache, invalidation |
| **Ambiguity** | FR-AMBIG-001 to FR-AMBIG-003: Handle multiple matches, prefer exact match |
| **Missing Fields** | FR-MISSING-001 to FR-MISSING-004: Graceful degradation, strict mode, logging |
| **Testing** | FR-TEST-001 to FR-TEST-003: Mock resolver, offline testing support |
| **Performance** | < 50ms cold resolution, < 1ms cached, zero extra API calls |

## System Context

### Problem Analysis

#### Current State

```
Schema Definition (unit.py):
  ColumnDef(name="mrr", source=None)  <-- No way to resolve
                        |
                        v
custom_fields.py:
  MRR_GID = "PLACEHOLDER_MRR_GID"     <-- Static, environment-specific
                        |
                        v
Extractor:
  value = extract_by_gid(task, MRR_GID)  <-- Manual wiring required
```

**Problems:**
1. 16 static GID placeholders require manual population per environment
2. `source=None` in schemas provides no information for extraction
3. No self-documenting relationship between field names and Asana fields
4. Doesn't scale: each new field requires code changes in 3 places

#### Desired State

```
Schema Definition (unit.py):
  ColumnDef(name="mrr", source="cf:MRR")  <-- Self-documenting
                        |
                        v
CustomFieldResolver:
  resolve("MRR") -> find in task.custom_fields by normalized name
                        |
                        v
Extractor:
  value = resolver.get_value(task, "mrr")  <-- Automatic resolution
```

**Benefits:**
1. Zero static GID constants needed
2. Self-documenting schemas (source tells you what to look for)
3. Environment-agnostic (GIDs discovered at runtime)
4. Scales to any number of fields without code changes

### Data Model

#### ColumnDef.source Convention

The `source` attribute uses a prefix convention to indicate resolution strategy:

| Prefix | Meaning | Example |
|--------|---------|---------|
| `None` | Use column name as custom field name | `ColumnDef(name="mrr", source=None)` |
| `"cf:{name}"` | Explicit custom field name | `ColumnDef(name="mrr", source="cf:MRR")` |
| `"gid:{gid}"` | Explicit GID (bypass resolution) | `ColumnDef(name="mrr", source="gid:123456")` |
| No prefix | Attribute path on task model | `ColumnDef(name="created", source="created_at")` |

from autom8_asana.models.task import Task

class CustomFieldResolver(Protocol):

    using the task's existing custom_fields list.

#### NameNormalizer

```python
# autom8_asana/dataframes/resolver/normalizer.py

import re
from functools import lru_cache

class NameNormalizer:
    """Normalize field names for matching.

    Converts various naming conventions to a canonical form
    for comparison:
    - "Weekly Ad Spend" -> "weeklyadspend"
    - "weekly_ad_spend" -> "weeklyadspend"
    - "MRR" -> "mrr"
    - "Monthly-Recurring-Revenue" -> "monthlyrecurringrevenue"
    """

    @staticmethod
    @lru_cache(maxsize=1024)
    def normalize(name: str) -> str:
        """Normalize name to canonical form.

        Args:
            name: Field name in any format

        Returns:
            Canonical lowercase form with only alphanumeric chars
        """
        if not name:
            return ""
        # Remove non-alphanumeric, lowercase
        return re.sub(r'[^a-z0-9]', '', name.lower())

    @staticmethod
    def is_match(name1: str, name2: str) -> bool:
        """Check if two names match after normalization."""
        return NameNormalizer.normalize(name1) == NameNormalizer.normalize(name2)
```

#### DefaultCustomFieldResolver

```python
# autom8_asana/dataframes/resolver/default.py

import threading
from typing import Any
from decimal import Decimal

from autom8_asana.models.task import Task
from autom8_asana.models.custom_field import CustomField
from .normalizer import NameNormalizer


class DefaultCustomFieldResolver:
    """Default implementation of CustomFieldResolver.

    Builds a name->gid index from the first task's custom_fields,
    then uses that index for all subsequent lookups.

    Thread-safe for concurrent extraction.
    """

    def __init__(self, strict: bool = False):
        """Initialize resolver.

        Args:
            strict: If True, raise on missing fields. If False, return None.
        """
        self._strict = strict
        self._index: dict[str, str] = {}  # normalized_name -> gid
        self._gid_to_info: dict[str, dict[str, Any]] = {}  # gid -> field info
        self._built = False
        self._lock = threading.RLock()

    def build_index(self, custom_fields: list[CustomField]) -> None:
        """Build name->gid index from custom fields.

        Only builds once; subsequent calls are no-ops.

        Args:
            custom_fields: List of CustomField objects from task
        """
        if self._built:
            return

        with self._lock:
            if self._built:
                return

            for cf in custom_fields:
                if cf.gid and cf.name:
                    normalized = NameNormalizer.normalize(cf.name)
                    self._index[normalized] = cf.gid
                    self._gid_to_info[cf.gid] = {
                        "name": cf.name,
                        "type": cf.resource_subtype,
                        "normalized": normalized,
                    }

            self._built = True

    def resolve(self, field_name: str) -> str | None:
        """Resolve field name to GID.

        Handles prefixes:
        - "cf:Name" -> resolve "Name"
        - "gid:123" -> return "123" directly
        - "name" -> resolve "name"

        Args:
            field_name: Field name with optional prefix

        Returns:
            GID if resolvable, None otherwise
        """
        # Handle gid: prefix (explicit GID)
        if field_name.startswith("gid:"):
            return field_name[4:]

        # Handle cf: prefix (explicit custom field name)
        if field_name.startswith("cf:"):
            lookup_name = field_name[3:]
        else:
            lookup_name = field_name

        normalized = NameNormalizer.normalize(lookup_name)
        return self._index.get(normalized)

    def get_value(
        self,
        task: Task,
        field_name: str,
        expected_type: type | None = None,
    ) -> Any:
        """Extract custom field value from task.

        Args:
            task: Task to extract from
            field_name: Schema field name (with optional prefix)
            expected_type: Optional type for coercion

        Returns:
            Extracted value (coerced if type provided), or None

        Raises:
            KeyError: If strict mode and field not found
        """
        gid = self.resolve(field_name)

        if gid is None:
            if self._strict:
                raise KeyError(f"Cannot resolve custom field: {field_name}")
            return None

        # Find custom field in task by GID
        for cf in task.custom_fields or []:
            if cf.gid == gid:
                raw_value = self._extract_raw_value(cf)
                if expected_type and raw_value is not None:
                    return self._coerce(raw_value, expected_type)
                return raw_value

        return None

    def has_field(self, field_name: str) -> bool:
        """Check if field is resolvable."""
        return self.resolve(field_name) is not None

    def _extract_raw_value(self, cf: CustomField) -> Any:
        """Extract raw value based on custom field type."""
        match cf.resource_subtype:
            case "text":
                return cf.text_value
            case "number":
                return cf.number_value
            case "enum":
                return cf.enum_value.name if cf.enum_value else None
            case "multi_enum":
                return [opt.name for opt in cf.multi_enum_values or []]
            case "date":
                return cf.date_value.get("date") if cf.date_value else None
            case "people":
                return [p.gid for p in cf.people_value or []]
            case _:
                return cf.display_value

    def _coerce(self, value: Any, expected_type: type) -> Any:
        """Coerce value to expected type."""
        if value is None:
            return None

        try:
            if expected_type == Decimal and isinstance(value, (int, float)):
                return Decimal(str(value))
            if expected_type == str:
                return str(value)
            if expected_type == int:
                return int(value)
            if expected_type == float:
                return float(value)
            if expected_type == list and not isinstance(value, list):
                return [value]
        except (ValueError, TypeError):
            return None

        return value

    def get_resolution_stats(self) -> dict[str, Any]:
        """Get statistics about resolved fields (for debugging)."""
        return {
            "indexed_fields": len(self._index),
            "field_names": list(self._gid_to_info.values()),
        }
```

#### Extractor Integration

```python
# autom8_asana/dataframes/extractors/base.py

from autom8_asana.dataframes.resolver.protocol import CustomFieldResolver

class BaseExtractor:
    """Base extractor with custom field resolution support."""

    def __init__(
        self,
        schema: DataFrameSchema,
        resolver: CustomFieldResolver | None = None,
    ):
        self._schema = schema
        self._resolver = resolver or DefaultCustomFieldResolver()

    def extract(self, task: Task) -> TaskRow:
        """Extract task to row using resolver for custom fields."""
        # Ensure index is built on first task
        self._resolver.build_index(task.custom_fields or [])

        row_data = {}
        for col in self._schema.columns:
            row_data[col.name] = self._extract_column(task, col)

        return self._create_row(row_data)

    def _extract_column(self, task: Task, col: ColumnDef) -> Any:
        """Extract single column value."""
        if col.extractor:
            # Custom extractor function
            return col.extractor(task)

        if col.source is None:
            # No source - derived field, subclass handles
            return None

        if col.source.startswith("cf:") or col.source.startswith("gid:"):
            # Custom field - use resolver
            return self._resolver.get_value(
                task,
                col.source,
                expected_type=self._get_python_type(col.dtype),
            )

        # Attribute path on task
        return self._get_attribute(task, col.source)
```

### Data Flow

#### Resolution Flow

```
+------------------------------------------------------------------------------+
|                           RESOLUTION FLOW                                     |
+------------------------------------------------------------------------------+

  First Task
       |
       | task.custom_fields = [
       |   {gid: "123", name: "MRR", type: "number", number_value: 5000},
       |   {gid: "456", name: "Weekly Ad Spend", type: "number", ...},
       |   ...
       | ]
       |
       v
+------------------------------------------------------------------------------+
| resolver.build_index(task.custom_fields)                                     |
|                                                                              |
|   For each custom field:                                                     |
|     normalized = NameNormalizer.normalize(cf.name)                           |
|     _index[normalized] = cf.gid                                              |
|                                                                              |
|   Result:                                                                    |
|     _index = {                                                               |
|       "mrr": "123",                                                          |
|       "weeklyadspend": "456",                                                |
|       ...                                                                    |
|     }                                                                        |
+------------------------------------------------------------------------------+
       |
       | Index built (thread-safe, once per session)
       v
  Subsequent Tasks
       |
       | Extraction for ColumnDef(name="mrr", source="cf:MRR")
       v
+------------------------------------------------------------------------------+
| resolver.get_value(task, "cf:MRR", Decimal)                                  |
|                                                                              |
|   1. resolve("cf:MRR")                                                       |
|      - Strip prefix: "MRR"                                                   |
|      - Normalize: "mrr"                                                      |
|      - Lookup: _index["mrr"] -> "123"                                        |
|                                                                              |
|   2. Find cf with gid="123" in task.custom_fields                            |
|      - cf.number_value = 5000.0                                              |
|                                                                              |
|   3. Coerce to Decimal                                                       |
|      - Decimal("5000.0")                                                     |
|                                                                              |
|   Return: Decimal("5000.0")                                                  |
+------------------------------------------------------------------------------+
```

#### Ambiguity Handling Flow

```
+------------------------------------------------------------------------------+
|                        AMBIGUITY HANDLING                                     |
+------------------------------------------------------------------------------+

  Scenario: Multiple fields normalize to same key

  task.custom_fields = [
    {gid: "111", name: "MRR"},
    {gid: "222", name: "mrr"},  # Duplicate after normalization
  ]

  Resolution Strategy:

  +----------------------------------------------------------------------------+
  | 1. First match wins (stable ordering)                                      |
  |    _index["mrr"] = "111"  # First one encountered                          |
  |                                                                            |
  | 2. Log warning for duplicate                                               |
  |    logger.warning("Duplicate custom field name", name="mrr", gids=[...])   |
  |                                                                            |
  | 3. If exact name match exists, prefer it                                   |
  |    resolve("mrr") checks for exact match before normalized                 |
  |    If both "MRR" and "mrr" exist, resolve("MRR") -> exact match           |
  +----------------------------------------------------------------------------+
```

## Technical Decisions

| Decision | Choice | Rationale | ADR |
|----------|--------|-----------|-----|
| Resolution strategy | Session-cached index + Protocol DI | Best of both worlds: fast lookups + testability | [ADR-0034](../decisions/ADR-0034-dynamic-custom-field-resolution.md) |
| Name normalization | Canonical form (lowercase, alphanumeric only) | Handles all naming conventions bidirectionally | [ADR-0034](../decisions/ADR-0034-dynamic-custom-field-resolution.md) |
| Integration point | Standalone resolver component | Single responsibility, testable in isolation | [ADR-0034](../decisions/ADR-0034-dynamic-custom-field-resolution.md) |
| DI pattern | Protocol-based | Follows ADR-0001, enables mock injection | [ADR-0001](../decisions/ADR-0001-protocol-extensibility.md) |

## Risks & Mitigations

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| Name collisions after normalization | Medium | Low | Log warnings, prefer exact match |
| Field name changes in Asana | Medium | Low | Document expected names in schema |
| Performance regression | Low | Low | LRU cache on normalizer, index once per session |
| Thread-safety issues | High | Low | Use RLock for index building |
