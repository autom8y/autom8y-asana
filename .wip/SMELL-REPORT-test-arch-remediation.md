## Category 1: CI Pipeline Architecture

### SM-001: fast-tests runs lint + typecheck + ALL unit tests sequentially (CRITICAL)

**Category**: CI Architecture
**Location**: `.github/workflows/test.yml:16-82`

**Evidence**: The `fast-tests` job runs in sequence:
1. `ruff format . --check` (line 65)
2. `ruff check .` (line 68)
3. `mypy src/autom8_asana --strict` (line 71)
4. `pytest tests/ -m "not slow and not integration and not benchmark"` (line 74)

All four steps run serially in a single job. Lint and typecheck are pure static analysis with no test dependency. Separating them into a parallel job would save the 3-5 minutes they consume from the test timeout budget.

**Blast Radius**: Every PR, every push -- this is the critical path
**Fix Complexity**: Low (split into 2-3 parallel jobs)
**ROI Score**: 10/10

### SM-002: full-tests re-runs ALL fast tests plus slow tests (CRITICAL)

**Category**: CI Architecture / Duplication
**Location**: `.github/workflows/test.yml:83-139`

**Evidence**:
- `fast-tests` (line 74): `pytest tests/ -m "not slow and not integration and not benchmark"`
- `full-tests` (line 133): `pytest tests/ -m "not integration"`

The `full-tests` job re-runs every test that `fast-tests` already ran, plus the 24 slow-marked tests. With 10,662 non-slow tests and only 24 slow tests, full-tests duplicates 99.8% of fast-tests work. The 24 slow tests are the only additive value.

**Blast Radius**: Every push to main, every scheduled run
**Fix Complexity**: Low (change full-tests to `-m "slow and not integration"`)
**ROI Score**: 10/10
