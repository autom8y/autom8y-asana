# WS-P2-01: Utility Extraction + EntityRegistry Fix

**Objective**: Extract two utility functions trapped in service/persistence layers
to `core/`, and register EntityRegistry with SystemContext to fix a test isolation bug.

**Rite**: hygiene
**Complexity**: PATCH
**Gaps**: G-01 (to_pascal_case), G-02 (register_holder), G-03 (EntityRegistry)
**Preconditions**: None (independent, do first)
**Estimated Effort**: 2 hours

---

## Problem

Two utility functions are trapped in higher-layer modules, creating unnecessary
cross-layer imports:

1. `to_pascal_case()` lives in `services/resolver.py` but is used by `cache/`,
   `core/`, and `dataframes/` -- all lower layers that should not import from services.
2. `register_holder()` lives in `persistence/holder_construction.py` but is called
   by 9 `models/business/*.py` modules -- models should not import from persistence.
3. `EntityRegistry` in `core/entity_registry.py` lacks `register_reset()`, causing
   state leaks between tests.

---

## Implementation Sketch

### G-01: Extract `to_pascal_case()` to `core/string_utils.py` (30 min)

1. Create `src/autom8_asana/core/string_utils.py` with `to_pascal_case()` (copy from `services/resolver.py`)
2. Replace the function body in `services/resolver.py` with:
   `from autom8_asana.core.string_utils import to_pascal_case`
   (keep re-export for any external callers)
3. Update 5 import sites to import from `core.string_utils`:
   - `cache/dataframe/factory.py:55` (runtime)
   - `cache/integration/schema_providers.py:27` (runtime)
   - `core/registry_validation.py:62` (deferred -> runtime)
   - `core/schema.py:70` (deferred -> runtime)
   - `dataframes/builders/progressive.py:70` (deferred -> runtime)
4. Run: `pytest tests/unit/cache/ tests/unit/core/ tests/unit/dataframes/ -x`
5. Gate: `grep "from autom8_asana.services.resolver import to_pascal_case" src/` -> 0 matches

### G-02: Move `register_holder()` to `core/registry.py` (45 min)

1. Create `src/autom8_asana/core/registry.py` with `register_holder()` (copy from `persistence/holder_construction.py`)
2. Keep re-export in `persistence/holder_construction.py` for backward compat
3. Update 9 import sites in `models/business/*.py`:
   - `contact.py`
   - `unit.py`
   - `offer.py`
   - `location.py`
   - `process.py`
   - `business.py`
   - `custom_field_accessor.py`
   - (verify 2 more by grep: `grep "from autom8_asana.persistence.holder_construction import register_holder" src/ -r`)
4. Run: `pytest tests/unit/persistence/ tests/unit/models/ -x`
5. Gate: `grep "from autom8_asana.persistence.holder_construction import register_holder" src/autom8_asana/models/` -> 0 matches

### G-03: Register EntityRegistry with SystemContext (15 min)

1. Read `src/autom8_asana/core/entity_registry.py`
2. Add `reset()` classmethod if not already present (clear internal state)
3. Add at module bottom:
   ```python
   from autom8_asana.core.system_context import register_reset
   register_reset(EntityRegistry.reset)
   ```
4. Run: `pytest tests/unit/core/ -x`
5. Gate: `grep "register_reset" src/autom8_asana/core/entity_registry.py` -> 1 match

### Final QA Gate

- Full suite: `AUTOM8Y_ENV=production .venv/bin/python -m pytest tests/ -x`
