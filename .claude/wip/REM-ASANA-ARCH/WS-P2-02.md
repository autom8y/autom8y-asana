# WS-P2-02: Surgical Cycle Cuts (5 Bidirectional Imports)

**Objective**: Break 5 bidirectional 2-cycles by relocating imports, extracting shared
types, or converting to protocol-based indirection. Each cycle has a specific fix
pattern; no cycle requires major redesign.

**Rite**: 10x-dev
**Complexity**: MODULE
**Gaps**: G-04 (5 surgical cycles)
**Preconditions**: P2-01 (soft -- G-01 helps core<->dataframes cut)
**Estimated Effort**: 4-5 hours

---

## Problem

13 bidirectional 2-cycles exist at the package level. 5 are surgical (fixable by
relocating 1-3 imports each). The other 8 are structural coupling (clients<->persistence,
cache<->dataframes, etc.) requiring major redesign -- skip those.

**Target cycles** (ordered smallest to largest):

| Cycle | Direction to Break | Imports to Move |
|-------|-------------------|-----------------|
| automation <-> lifecycle | lifecycle->automation (1 import) | 1 |
| dataframes <-> services | services->dataframes (2 imports) in progressive.py | 2 |
| cache <-> models | models->cache (1 import) in detection/facade.py | 1 |
| core <-> dataframes | core->dataframes (1 import) in schema.py | 1 |
| core <-> models | core->models (2 imports) in registry_validation.py + schema.py | 2 |

---

## Implementation Sketch

**Order**: smallest-to-largest to build confidence and validate patterns.

### Cycle 1: automation <-> lifecycle (15 min)

**Files**:
- `automation/workflows/pipeline_transition.py:194` -- imports from lifecycle
- `lifecycle/seeding.py:29` -- imports from automation

**Fix**: The lifecycle->automation import in `seeding.py` is for a shared type or
constant. Extract it to `core/` or make it a parameter.

1. Read both files to identify the exact import
2. Extract shared type/constant to `core/pipeline_types.py` or similar
3. Update both import sites
4. Gate:
   ```bash
   grep "from autom8_asana.lifecycle" src/autom8_asana/automation/ | wc -l  # -> 0
   grep "from autom8_asana.automation" src/autom8_asana/lifecycle/ | wc -l  # -> 0
   ```
5. Run: `pytest tests/unit/automation/ tests/unit/lifecycle/ -x`

### Cycle 2: dataframes <-> services (30 min)

**Files**:
- `dataframes/builders/progressive.py:1234-1235` -- imports 2 items from services

**Fix**: The dataframes module should not import from services. Extract the 2 service
references to a callback parameter or protocol.

1. Read `dataframes/builders/progressive.py` around lines 1234-1235 to identify imports
2. If imports are for type hints only: move to TYPE_CHECKING
3. If runtime: extract as callback parameter or use a protocol
4. Gate:
   ```bash
   grep "from autom8_asana.services" src/autom8_asana/dataframes/ | wc -l  # -> 0
   ```
5. Run: `pytest tests/unit/dataframes/ -x`

### Cycle 3: cache <-> models (30 min)

**Files**:
- `cache/integration/derived.py:19` -- imports from models (expected direction)
- `models/business/detection/facade.py:24` -- imports from cache (wrong direction)

**Fix**: The model should not import from cache. Replace with a protocol or move
the detection concern.

1. Read `models/business/detection/facade.py` around line 24
2. Identify what is imported from cache
3. If it is a type reference: move to TYPE_CHECKING guard
4. If runtime: extract a protocol in `protocols/` or `core/` that cache satisfies
5. Gate:
   ```bash
   grep "from autom8_asana.cache" src/autom8_asana/models/ | wc -l  # -> 0
   ```
6. Run: `pytest tests/unit/models/ tests/unit/cache/ -x`

### Cycle 4: core <-> dataframes (45 min)

**Files**:
- `core/schema.py:26` -- imports from dataframes (wrong direction)
- Many dataframes -> core imports (expected direction, keep)

**Fix**: core/ should NOT import from dataframes/. Move the schema utility that
depends on dataframes out of core, or extract the needed type to core.

**Note**: If P2-01 completed G-01 (`to_pascal_case` to `core/string_utils.py`),
some deferred imports from core->dataframes may already be reduced. Verify before
making changes.

1. Read `core/schema.py` around line 26 to identify the dataframes import
2. Determine if it is for a type, a function, or data
3. Move the imported item to core (if it belongs there) or reverse the dependency
4. Gate:
   ```bash
   grep "from autom8_asana.dataframes" src/autom8_asana/core/ | wc -l  # -> 0
   ```
5. Run: `pytest tests/unit/core/ tests/unit/dataframes/ -x`

### Cycle 5: core <-> models (1 hr)

**Files**:
- `core/registry_validation.py:139` -- imports from models
- `core/schema.py:26` -- imports from models

**Fix**: core/ should NOT import from models/. Extract the model types used by
core into `core/types.py` or use protocols.

1. Read both files to identify what is imported from models
2. If types/constants: extract to `core/types.py`
3. If runtime functions: use protocol indirection

5. Gate:
